Overview

This repository contains a Power BI report built on the Global Superstore dataset. The dashboard includes KPI cards, trend lines, bar charts, a donut chart for category share, and interactive slicers for City, Order Priority and Product Category.

What the report shows

  • Total Sales, Total Profit, Total Cost and Profit %
  • Sales by Customer Segment and by Product Category
  • Profit % by Product Container
  • Sales & Profit trend across time
  • Interactive filters and Last Refresh timestamp

Data Model & ETL

Power Query transformations applied:

  • Converted date columns to Date/DateTime
  • Created a dedicated Date (Calendar) table and marked as date table
  • Standardized category names and removed nulls
  • Computed Total Shipping Cost (if missing) and added Net Profit columns

Model design

  • Fact: Orders (Sales, Profit, Shipping Cost)
  • Dimensions: Calendar, Customer, Product, Region
  • Primary relationships: Order Date ↔ Calendar[Date], Product ↔ Product Table

Key DAX measures

Total Sales = SUM(Orders[Sales])

Total Profit = SUM(Orders[Profit])

Profit % = DIVIDE(SUM(Orders[Profit]), SUM(Orders[Sales]))

Net Profit = SUM(Orders[Profit]) - SUM(Orders[Total Shipping Cost])

Net Profit % = DIVIDE([Net Profit], SUM(Orders[Sales]))

LastRefresh = LASTREFRESH()

How to open & use

  1. Download the dataset using the Download Sample Dataset button above.
  2. Open Superstore Power BI Report.pbix in Power BI Desktop.
  3. In Power Query, point the Orders table to the dataset CSV if needed and refresh.
  4. Verify relationships and the Date table is marked as the model date table.
  5. Interact with slicers, KPI cards and check the Last Refresh timestamp.

Tip: To enable on-demand dataset refresh from the report, use the Power Automate visual and create a flow that calls Refresh a dataset in Power BI Service (requires appropriate permissions).
